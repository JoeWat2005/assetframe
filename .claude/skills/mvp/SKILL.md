# AssetFrame MVP Report Pipeline (/mvp INSTRUMENT)

Brand: **AssetFrame** — *"Next-session market intelligence, scored after the fact."*
Products: **AssetFrame Snapshot** (free, 1 page) and **AssetFrame Pro** (paid, 3–6 pages).
Positioning: general market research and decision support — never personal advice, never guaranteed, no execution. Logo (`logo/logo_trimmed.png`, wordmark "Asset"+boxed "Frame") appears on every PDF header, HTML export, and preview. Report IDs `AF-YYYYMMDD-<INSTRUMENT>-FREE|-PRO`.

Output per run: `reports/YYYY-MM-DD/<INSTRUMENT>/` → `free.pdf`, `pro.pdf`, `free.html`, `pro.html`, `metadata.json`, `preview.png`.

Data layout: `data/candles/` (hourly+daily CSVs) · `data/analysis/` (engine JSON) · `data/payloads/` (report payloads) · `data/predictions/` (registered predictions) · `ledger/outcome_ledger.csv` (append-only track record).

Language: use Research view / Long-biased scenario / Short-biased scenario / Conditional setup / Invalidation / No-trade condition / Scenario / General market research / Not personal advice. NEVER write: you should buy/sell, sure trade, risk-free, easy profit. "Guaranteed" and "personal recommendation" are allowed ONLY in negated compliance form ("No outcome is guaranteed", "not a personal recommendation") - the QA gate checks the preceding context. R:R only as "T1 1.5x; T2 2.1x" / "T1 below 1.0x; T2 1.4x" / "No valid R:R - excluded". Authoring trap: never place "~" directly before a negative number ("~-40%") - the negative-R:R lint flags it; write "down ~40%". The generator's QA gate hard-fails the build on violations. The Free teaser and disclaimer fields are exempt from the free-split content scan (the teaser legitimately NAMES Pro features); everything else in the free tier is scanned.

## The 13-step flow

1. **Score expired ledger windows first** — check `data/predictions/*_predictions.json` for passed `window_end_utc` with no `ledger/outcome_ledger.csv` row; resolve `manual` predictions via WebSearch (leave genuinely unresolvable ones MANUAL with a stated reason); refresh that instrument's hourly CSV; run `python scripts/score_report.py <file> [--manual ID=Y|N|NT]`. Ledger is append-only; never score an incomplete window; calibration block appears at ≥10 rows.
2. **Run the engine**: `python scripts/intraday.py <YAHOO_SYMBOL> --name <PREFIX> --hrange 10d --roll-utc 22 --related "..."` (roll-utc 22 for futures/FX/crypto; CSVs land in `data/candles/`, analysis JSON in `data/analysis/`). Read `freshness`, `degraded`, `windows` before using any number. Pick `--related` for the asset class (include ^VIX,^VVIX,^VIX3M for equity indices — they feed the options-context section).
3. **Gather context**: news/catalysts (WebSearch, official sources first), economic calendar with exact UK times, positioning (CFTC/COT), options context where sourceable (VIX family for indices; CoinGecko derivatives for crypto; otherwise "unavailable"), cross-checks for the live price (CoinGecko for crypto; independent web quotes otherwise). **Sentiment (data-gated)**: for crypto fetch the Fear & Greed index (`https://api.alternative.me/fng/` — keyless) and funding/OI/basis; for any asset add social-sentiment reads (e.g. `last30days` Reddit/HN/Polymarket) when genuinely sourceable. Every sentiment row needs source + timestamp; if nothing is sourceable, omit `pro.sentiment` entirely — never filler.
4. **Apply session rules**: `python scripts/sessions.py <profile>` or import `scripts/sessions.get_session` — profiles: `cme_futures`, `fx_spot`, `crypto_24_7`, `us_equity_rth` (single stocks/ETFs: pre-market 08:00-13:30 UTC / regular 13:30-20:00 UTC / after-hours 20:00-00:00 UTC in the EDT regime; the window targets the NEXT REGULAR session; engine runs WITHOUT `--roll-utc`; tradable levels are regular-session unadjusted prices; stock tone — Nasdaq session/pre-market/after-hours/earnings window/gap risk, never Globex/roll/maintenance language). It returns market state, open/close UTC, next maintenance break, and the **prediction window** (next full session when <90 min remain, <240 min on Fridays; rolling 24h for crypto — never say "market close" for crypto). Copy its fields into `meta.*` and its prose into the Pro "Asset-session rules" section. Web-verify exchange holidays when within a week of one; record in metadata.
5. **Build the canonical payload** (`data/payloads/<PREFIX>_af_payload.json`): every price that appears anywhere lives ONCE in `canonical.levels` (id, value, label, cls ∈ tail|resistance|target|trigger|entry|support|invalidation); `canonical.last_price` is the full price object (value/symbol/source/timestamp_utc/exchange_tz/bar_interval/bar_complete/price_type/adjustment/contract_month); `canonical.setups[]` carry entry_lo/entry_hi/invalidation/t1/t2 + `rr` string in the approved format; `canonical.ladder` lists level ids; `canonical.ledger_levels` lists EVERY numeric a prediction references - including manual/P6 reference prices such as "Friday's close" (declare them as levels, e.g. cls "support", even if they skip the ladder; the QA gate refuses the build otherwise). Setup quality labels ∈ {High quality, Acceptable, Low quality, Management only, No-trade}. If Friday's session is still trading when authoring a next-session report, label Friday-derived pivots "provisional — finalises at the Friday settle". Payload also carries: `pro.overview` (MANDATORY: 3–5 short plain-English lines, zero jargon — what's happening, what we're watching, what kills the view; rendered as the "In plain English" box and QA-warned if missing) and optional `pro.sentiment` = `{fear_greed: {value 0-100, label, source, asof}, rows: [[source, reading, why-it-matters], ...], note}` (rendered as a gauge diagram + table in "Sentiment & positioning"). Sections may carry `items: [{label, text}, ...]` instead of/alongside `html` — the PREFERRED form for Market summary and other label-led bullet lists (renders as label-break bullets in both PDF and HTML).
6. **Generate Snapshot** — included: logo, instrument+ticker+asset class, timestamp+TZ, risk window, status & risk badges, last price + bar time, broad expected range, basic data quality, ONE chart (≤3 labelled levels, no pivots/bands), 3 bullets (core thesis / main catalyst / main risk), simplified Bull/Base/Bear matrix, visual timeline strip, Pro teaser, short disclaimer. EXCLUDED (QA-enforced): entries, invalidation logic, R:R, sizing math, options ideas, scorecard, audit, ledger, price ladder.
7. **Generate Pro** — executive header (incl. primary bias + both setup summaries + key invalidation + key risk), daily regime chart, intraday chart, RSI, **price ladder** (upper tail → R2 → T2 → T1 → trigger → last → entry zone → support → invalidation → lower tail; only canonical levels), confidence gauge, then sections: Market summary (+cross-asset table), **Long / Short Research View** (primary bias; best long + best short scenario; activations; invalidations; both-sides no-trade; which side better R:R; which side more catalyst-sensitive; "No valid long or short setup - stand aside." when true), Scenario matrix (trigger/move/invalidation/confidence/watch), Event-risk timeline (event/time/TZ/relevance/in-window/changes-no-trade-rules/gap-risk), Technicals & key levels (distance + classification columns), Conditional setups (R:R format + expiry + notes), **Options / Hedging Context** (data-gated: VIX/VVIX/term structure & implied-move-vs-ATR for indices; venue IV/funding/basis for crypto if sourced; otherwise exactly "No options context included: reliable IV/skew data unavailable." — never construct option trades, never "buy/sell this option"), **Asset-Specific Statistics** (per-class menu below; every stat has source+timestamp+confidence; no filler), "What can go wrong?", generic contract/risk math (educational, with the sizing-depends-on-circumstances line), Trade-quality scorecard (8 components /10 + final confidence), **Outcome ledger** (ID/Claim/Window/Trigger/Scoring rule/Result-Pending; "Ledger starts here." if empty; Free may MENTION scoring but never shows the ledger), full Source audit (provider, bar timestamp+age, cross-check, lookback daily+intraday, warm-up confirmed, partial-indicators-hidden, gaps, quota failures, single-source claims, DQ score), Asset-session rules, footer on every page.
8. **metadata.json** — full schema (see `scripts/mvp_report.py` build_metadata + the payload meta block): brand/tagline/products, venue + exchange_timezone + report_timezone (Europe/London), dual-TZ generated_at and window bounds, session fields, latest bar + completeness, data_provider + cross_check_provider, price_type/contract_month/adjustment, status/risk/research_view/primary_bias/scenario qualities, data_quality_score, warm-up flags, lookback_used, asset_specific_stats_included[], options_context_included+reason, source_gaps[], high_impact_claims[], qa_checks{}, paths{}.
9. **preview.png** — page 1 of free.pdf (pymupdf, ~130dpi). Generated automatically.
10. **Register Pro predictions** — write `data/predictions/<PREFIX>_predictions.json` (scorer schema; `hourly_csv` pointing at `data/candles/<PREFIX>_hourly.csv`) for the report's window so the next run scores it. If the instrument's prior predictions file is still UNSCORED (its window has not closed), do NOT overwrite it - write `<PREFIX>_af_predictions.json` instead; both names match the scorer's `*_predictions.json` scan and the ledger's report_id check prevents double-scoring. Every ledger row in the PDF must match the registered file.
11. **QA checks** — automatic in the generator (build aborts on failure): price triple-equality (CSV last == canonical == header), levels↔setups↔ladder↔ledger identity, R:R lint, banned-language scan, free/pro split, claim gating (unverified claims cannot be used_in_thesis), timestamps UTC-normalized, no lookahead, session fields present, logo present.
12. **Visual inspection** — Read free.pdf, pro.pdf, preview.png (and spot-check HTML) page by page against: logo present, no placeholder branding, no overlap/clipping/cramping, separate status & risk badges, no annotation collisions, simple Free chart, ladder present in Pro and matching tables, readable audit + setups, unambiguous R:R, warmed indicators, session rules applied, options context omitted where unsupported, claims gated, correct paths. Then `python scripts/mvp_report.py <out_dir> --stamp-visual`.
13. **Return** paths + changelog + fixed issues + remaining data gaps + QA checklist + the confirmations (Free/Pro distinct, logo used, claims gated, stats included/omitted deliberately, options context only where supported).

## Editorial polish (premium standard)

- **Market summary format**: author as structured `items` (`[{label: "Technical thesis", text: "Short sharp sentences..."}, ...]`) — renders as label-break bullets everywhere. Labels: Technical thesis / Macro-catalyst thesis / Cross-asset read / What would prove this wrong / Timing risk. No bullet over ~3 rendered lines; skimmable in under 20 seconds; institutional tone, never promotional.
- **NEVER lead a bullet, paragraph or label with "-"** — bullet glyphs come from the renderer. The generator strips leading dashes at structural starts (negative numbers are safe) and QA-warns; don't rely on it.
- **Emphasis that carries**: wrap a WHOLE table cell in `<b>` to make it bold in the PDF too; numeric columns right-align automatically; tables get header fill + zebra rows — author short cells and let the renderer do the look.
- **Auto-rendered blocks (never author these manually)**: key-levels chip strip (from `canonical.setups[0]`), ladder %-distances + legend, chart/levels glossary (from what the report actually uses), Fear & Greed gauge (from `pro.sentiment.fear_greed`).
- **Canonical Pro section order**: Market summary → Long/Short Research View → Scenario matrix → Event-risk timeline → Technicals & key levels → Conditional setups → Options/Hedging → Asset-Specific Statistics → What can go wrong? → contract/risk math → Trade-quality scorecard → Outcome ledger → Source audit → Asset-session rules. QA warns on known sections out of this order.
- **Pro verdict box** (mandatory, after the exec header, before charts): payload `pro.verdict` = `{line, best, risk, stand_aside}` — one concise conditional-scenario sentence, then Best opportunity / Main risk / Stand-aside condition. Never an instruction.
- **Source confidence box + Report quality card** (mandatory, rendered just before the Source audit section): payload `pro.source_confidence` and `pro.report_quality` as `[["Price data", "..."], ...]` pairs — price/catalyst/options/session/overall confidence, and data-quality + QA pass/fail lines. Both mirror into metadata.
- **"Why it matters" micro-explanations**: every stats/cross-asset table carries a short professional why-it-matters phrase per row (e.g. "VIX lower = less demand for downside protection"). Short phrases, never lectures.
- **Tables**: commentary below tables, short cell text, premium pagination (a section never starts in the bottom ~34mm of a page). Pro may run 4-7 pages — readable beats compact.
- **Price ladder**: short labels, consistent decimals, auto-caption "Levels are conditional research references, not trade instructions."

## High-impact claim gating

IPO/debut, market-cap, index inclusion/exclusion, central-bank probabilities, geopolitical deals, official inventories, earnings dates, major corporate/analyst/ratings news, CFTC positioning, exchange schedule changes, roll assumptions, options/gamma, ETF flows → each labelled **confirmed / multiple-source / single-source / unverified / stale / unavailable** in `meta.high_impact_claims` with source and `used_in_thesis`. Confirmed and multiple-source claims may drive thesis; single-source may support it (labelled, never central); **unverified/stale/unavailable must NOT drive thesis (generator hard-fails)**. Never overstate: write "multiple-source reports of a draft agreement; signature remains unconfirmed", not "confirmed draft" — "confirmed" only when an official source or market data truly supports it. Market-moving single-source claims reduce the catalyst-confidence line.

## Asset-specific statistics menus (include only what is sourced; no filler)

- **Indices/futures**: cash-vs-futures distinction, fair value (if available), VIX/VVIX/term structure, breadth (A/D, % above DMAs — usually unavailable: say so), sector leadership, put/call & gamma (if sourced), cash open/close + futures close, overnight gap risk, contract roll status.
- **Oil/energy**: WTI/Brent distinction + spread, front month + roll status, API/EIA timings + inventories, crack spreads, refinery utilisation, OPEC+ headlines, Baker Hughes, CFTC positioning, geopolitical supply risk, settlement/maintenance.
- **Metals**: futures-spot basis, DXY, real/nominal yields, Fed path, central-bank demand (if sourced), haven context, gold/silver ratio, silver industrial demand, CFTC positioning, COMEX inventory/OI (if available).
- **FX**: session H/L by Asia/London/NY, daily/weekly ATR, rate differential, central-bank pricing, CFTC positioning, IV/risk-reversals (if available), rollover window, weekend gap, intervention risk, macro calendar.
- **Crypto**: 24h/7d range, funding rate, open interest, perp basis, liquidation clusters (if sourced), ETF flows (if available), BTC dominance, stablecoin liquidity, venue/outage risk, weekend liquidity, funding windows.
- **Stocks/ETFs** (future): prev close, OHLC + gap %, volume vs 20d, ATR percentile, VWAP, RS vs sector, earnings date/timing + implied move, short interest, options IV/put-call, S/R distance, pre/after-hours separation.
- **Bonds/rates** (future): yield change, curve slope, futures/cash distinction, auctions, inflation/employment data, CB pricing, real yields, term premium (if sourced).

## Shared infrastructure

`scripts/intraday.py` (warm-up-extended data + `windows` block) · `scripts/report_pdf.py` (charts with warm-crop + partial-indicator disclosure, tables, wrapping) · `scripts/sessions.py` (session profiles + window policy) · `scripts/mvp_report.py` (generator + QA gate + `--stamp-visual`) · `scripts/score_report.py` (append-only ledger, `--manual`, `--dry-run`, calibration). Warm-up minimums: daily 1y display ← 2y fetch (SMA200+buffer); hourly 10d display ← 31d fetch (SMA50/RSI14/MACD+buffer); partial lines hidden or labelled; never infer trend from cold indicators.
